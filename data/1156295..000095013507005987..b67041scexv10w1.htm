Description of the BSI-II System:
The Client, Transaction Server, and Database
The system uses a three-tiered client/server architecture backed by a Sybase SQL database to provide access to information on more than nine million specimens. The first tier is the client which executes on the users local machine, and provides the user interface for the BSI-II system. The client is written in Java and is optimized for the Microsoft Windows environment. The client connects to the second tier, the transaction server, via the Internet. The transaction server is written in non-GUI Java and executes on a Linux machine located at IMS. The transaction server implements the interface between the client and the third tier, the SQL database. For the third tier the BSI-II system uses the Sybase Adaptive Server Enterprise SQL database executing on a SPARC station located at IMS inside a firewall. (Note: this will be changing to a Postgres database and Linux during 2005).
